Создание проекта в VS.

Рекомендуется использовать Visual Studio 2012, так как в ней меньше подсказок по коду, отсутствует автозакрытие скобок, автоформатирование и многие вещи, которые упрощают разработку. Таким образом, студенты учатся сами следить за правильностью, чистотой и красотой кода.

  1. Запустить Visual Studio 2019
  2. Create a new project
  3. В окне выбрать Console App(.Net Core)
  4. Нажимаем Next. В появившемся окне обязательно ввести название проекта и выбрать свою рабочую папку. Имя должно отражать суть созданного проекта - обязательно! Если не выбрать папку, проект "улетит" непонятно куда (скорее всего в папку другого ученика)
  5. Нажимаем Create.

Hello World

У всех программистов, когда они начинают изучать новый язык, есть традиция - писать программу, которая его приветствует. Называется эта программа Hello, World! Для запуска программы нажмите CTRL + F5 после того, как напишете весь код! 

						
							#define _CRT_SECURE_NO_WARNINGS
							#include <stdio.h>
							int main()
							{
								printf("Hello, world!\n");
								return 0;
							}

Разберем подробно

  • #define _CRT_SECURE_NO_WARNINGS - строчка, которая нужна для того, чтобы убрать некоторые ошибки связанные с некоторыми устаревшими функциями. Писать ее именно в таком виде!
  • #include <stdio.h> - подключение библиотеки, которая содержит базовый функционал языка СИ и некоторые функции ввода и вывода. Расшифровывается как Standart Input Output - стандартная библиотека ввода и вывода.
  • int main() - здесь main() - главная функция, с выполнения которой начинается работа любой программы в языке СИ. Если этой функции нет, то выполняться и запускаться программа не будет. int - тип данных, о функциях и их типах мы поговорим позже.
  • Открывающая { и зыкрывающая } фигурные скобки определяют начало и конец блоков кода. Например, в данной программе эти скобки определяют начало и конец блока main.
  • Функция printf предназначена для вывода текста на экран. То, что написано внутри в кавычках - аргумент функции, в данном случае - текст. В тексте стоит \n - это перевод текста на следующую строку.
  • return 0; - данная строка завершает работу программы.

Комментарии в коде

Код ваших программ обязательно нужно комментировать в процессе написания. Это делается для того, чтобы в будущем, когда Вы вернетесь к своему проекту или работе после отпуска, Вы могли быстро вспомнить что и для чего писали. Кроме того, писать комментарии полезно для коллег, с которыми Вы работаете.

Есть два типа комментариев - многострочный и однострочный. Однострочный комментарий выглядит так:

//Вот это однострочный комментарий. Программа не воспринимает этот текст как код вообще. /*Вот это многострочный комментарий.
Программа не воспринимает этот текст как код вообще.
Но такой комментарий должен заканчиваться вот этими символами - */

Чаще и больше пишите комментарии для себя и своих коллег - это правило хорошего тона для любого, кто хочет стать программистом!
